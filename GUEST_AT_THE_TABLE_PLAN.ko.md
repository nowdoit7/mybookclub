# Guest at the Table — 제품 및 엔지니어링 계획

> 이 문서는 `GUEST_AT_THE_TABLE_PLAN.md`의 한국어 번역본입니다. 구현 시에는
> 영문 원본과 함께 갱신합니다.

## 제품 구상

The Reading Table에는 현실에서 만날 수 없는 역사적 인물이나 문학 속 인물을
상상 속 게스트로 가끔 초대할 수 있습니다. 게스트는 단순한 유명인 챗봇도,
토론을 끝내 버리는 권위자도 아닙니다. 기존 독자 세 자리 중 하나에 앉아
확고한 해석 관점을 제시하고, 다른 독자와 마찬가지로 반론을 받습니다.

모임 예시:

- 아이작 뉴턴이 자연철학자의 관점으로 『삼체』 3부작을 읽습니다.
- 아리스토텔레스가 인과관계와 제1원리로 현대 과학 서적을 읽습니다.
- 『삼국지연의』에 묘사된 유비가 작품 속 충성, 정통성, 통치에 관한 선택을
  읽습니다.

이 등장은 드물고 특별하게 느껴져야 합니다. 기존 여덟 독자가 제품의 중심이며,
게스트는 기억에 남는 특별 이벤트입니다.

## 핵심 결정

네 번째 AI 독자는 추가하지 않습니다. 게스트는 해당 세션에서 같은 카테고리의
일반 독자 한 명을 대신합니다.

| 게스트 | 카테고리 | 이유 |
| --- | --- | --- |
| 아이작 뉴턴 | analytical | 증거, 작동 원리, 수학적 질서, 추론의 한계 |
| 아리스토텔레스 | contextual | 현대의 전제를 드러내는 역사적으로 먼 사고 체계 |
| 소설 속 유비 | contextual | 작품의 도덕적·정치적 세계 안에 자리 잡은 관점 |

따라서 모든 세션은 계속 emotional, analytical, contextual 독자를 정확히 한 명씩
보유합니다. 엔진, UI, 독서 노트 호출 수, 발언 예산, 모임 기록 구조도 바뀌지
않습니다.

## 사용자에게 보여줄 고지

모든 게스트 카드에는 `상상 속 게스트` 배지를 항상 표시합니다. 알렉스는 사회적
자기소개 전에 다음과 같은 짧은 고지를 합니다.

> 오늘의 상상 속 게스트는 아이작 뉴턴입니다. 그의 발언은 기록으로 남은 사상을
> 바탕으로 재구성한 해석이며, 뉴턴의 실제 견해가 아닙니다.

이 고지는 모델의 판단이 아니라 화면 표현 규칙입니다. 선택된 페르소나에 게스트
메타데이터가 있으면 코드가 삽입합니다.

반드시 지킬 규칙:

- MVP 게스트는 오래전에 사망한 역사적 인물이나 퍼블릭 도메인 문학 인물로
  한정하며, 살아 있는 유명인은 제외합니다.
- 인용문을 날조하거나 생성된 대사를 해당 인물의 실제 발언으로 표현하지 않습니다.
- 역사적 인물이 현대 작품을 실제로 읽었다고 주장하지 않습니다.
- 다른 독자나 작가보다 특별히 높은 권위를 부여하지 않습니다.
- 역사적 인물과 문학적 형상을 구분합니다. `『삼국지연의』 속 유비`를 단순히
  역사적 유비라고 표시하지 않습니다.
- 인물 재현의 근거는 검토된 정적 자료에서만 가져옵니다. 런타임 웹 검색 호출은
  추가하지 않습니다.

## 결정론적 등장 자격과 희귀도

게스트의 등장 여부는 모델이 결정하지 않습니다. 순수 코드 선택기가 다음 조건을
모두 요구합니다.

1. Live 모드이며 도서 상태가 `verified`입니다.
2. 검증된 정식 제목, 저자, 시리즈 또는 엄격히 제한한 주제가 정적 게스트 규칙과
   일치합니다.
3. 세션 seed가 결정론적 희귀도 판정을 통과합니다.
4. 자격이 있는 게스트 중 정확히 한 명만 선택되고 일반 독자 한 명만 교체됩니다.

초기 등장 확률은 자격을 만족한 도서 중 15%입니다. 재현 가능한 데모 seed는 확률
판정만 건너뛰며, 도서 자격 판정은 절대 건너뛰지 않습니다.

- `?seed=demo`는 현재의 일반 데모 독자 세 명을 그대로 유지합니다.
- `?seed=guest-newton`은 자격이 있고 검증된 작품에만 뉴턴을 등장시킵니다.
- 관계없는 책에 같은 seed를 사용하면 일반 독자 세 명을 안전하게 반환합니다.

첫 버전은 정식 제목과 저자의 정확한 별칭 목록을 사용합니다. 현재 도서 검증
계약에는 신뢰할 만한 주제 태그가 없으므로 넓은 의미 기반 주제 매칭은 이후로
미룹니다.

## 제안 데이터 구조

게스트 페르소나도 페르소나 데이터로 유지합니다. 엔진에는 게스트 전용 분기문을
넣지 않습니다.

```ts
interface GuestPortrayal {
  kind: "historical" | "literary";
  displayLabel: Record<AppLanguage, string>;
  disclosure: Record<AppLanguage, string>;
  portrayalBasis: string[];
  sourceUrls: string[];
  eligibility: {
    canonicalTitles?: string[];
    authors?: string[];
    workScopes?: BookScope[];
  };
  rarity: number;
}

interface PersonaCard {
  // 기존 필드
  guestPortrayal?: GuestPortrayal;
}
```

`REGULAR_PERSONAS`와 `GUEST_PERSONAS`는 별도의 카탈로그로 내보냅니다. 선택 함수는
`selectPersonas(seed, confirmedBook, mode)`가 되며 계속 정확히 세 개의
`PersonaCard`를 반환합니다.

## 프롬프트 계약

게스트의 독서 노트에는 일반 독자와 같은 도서 정보와 엄격한 스키마를 제공하고,
다음의 변경 불가능한 재현 규칙을 추가합니다.

- 미래를 실제로 알고 있었다고 주장하지 않고 기록으로 확인되는 세계관에서
  추론합니다.
- 옛 말투를 과장해 흉내 내기보다 현대 개념을 이해하기 쉬운 언어로 표현합니다.
- 기억, 인용, 만남, 반응을 역사적 사실처럼 지어내지 않습니다.
- 반론을 받으면 인정할 수 있는 오류 가능한 독자로 남습니다.
- 역사 강의를 하기보다 현재 책 자체를 이야기합니다.

모임 기록에서는 해당 발언자를 상상 속 게스트로 표시하며, 생성된 대사를 역사적
인용문으로 바꾸지 않습니다.

## 첫 번째 세로 기능 범위

추천 데모 게스트는 우주론, 물리 법칙, 과학적 추론이 중심인 소수의 검증된 작품에
등장하는 **아이작 뉴턴**입니다. 첫 번째 명시적 대상은 『삼체』 3부작입니다.
이미 가장 강한 실제 토론 테스트 자료가 있고, 영어권 심사위원도 이 조합의
의외성을 즉시 이해할 수 있기 때문입니다.

첫 번째 세로 기능에는 다음이 포함됩니다.

1. 선별된 게스트 예외와 고지 규칙을 SPEC에 반영합니다.
2. `GuestPortrayal` 계약과 엄격한 검증을 추가합니다.
3. 검토를 마친 뉴턴 페르소나 카드와 초상화 각 하나를 추가합니다.
4. 순수 함수로 자격, 희귀도, 교체, seed 로직을 구현합니다.
5. 항상 보이는 게스트 배지와 알렉스의 고지를 추가합니다.
6. 모든 구조 검사를 통과한 뒤 mock 세션을 검증하고 마지막에 Live 스모크 세션을
   한 번만 실행합니다.

유비는 자격 범위를 하나의 퍼블릭 도메인 문학 작품으로 제한할 수 있고 재현 대상을
정확히 명시할 수 있으므로 더 안전한 두 번째 게스트입니다. 처음 두 게스트가 단순히
유명한 외형만 제공하는 것이 아니라 해석을 깊게 만든다는 사실을 증명한 뒤
아리스토텔레스를 추가합니다.

## 필수 테스트

- 일반 세션은 계속 정확히 세 명과 카테고리별 한 명을 선택합니다.
- 게스트 데모 seed를 사용해도 관계없는 책에는 게스트가 등장하지 않습니다.
- 자격이 있는 같은 책과 같은 seed는 언제나 같은 세 명을 반환합니다.
- 한 세션에는 게스트가 최대 한 명만 등장합니다.
- 게스트는 자신의 카테고리에 속한 일반 독자만 교체합니다.
- `?seed=demo`는 기존 일반 데모 독자 세 명을 유지합니다.
- 게스트 프롬프트에 고지와 가짜 인용 금지 규칙이 포함됩니다.
- 게스트 대사는 2~4문장을 지키며 기존의 모든 반론, 책장, 모임 기록, 저작권
  불변식을 통과합니다.
- Mock 모드는 게스트가 검증되지 않은 책의 내용을 안다고 주장하지 않습니다.

## 구현 전에 해결할 SPEC 충돌

현재 SPEC은 페르소나 풀을 여덟 명으로, 초상화 리소스를 알렉스와 일반 독자 여덟
명으로, 추첨을 고정된 각 카테고리 풀에서 한 명씩 뽑는 방식으로 규정합니다.
구현 단계에서는 먼저 다음과 같이 해당 내용을 수정해야 합니다.

> 핵심 풀에는 일반 독자 여덟 명이 있습니다. 엄선한 상상 속 게스트가 드물게 같은
> 카테고리의 일반 독자 한 명을 대신할 수 있습니다. 모든 세션은 여전히 정확히 세
> 명의 AI 독자와 카테고리별 한 명으로 구성됩니다.

이 문서는 런타임 코드를 의도적으로 변경하지 않습니다. 별도 기능 브랜치에서
구현을 시작하기 전에 승인할 범위를 정하는 문서입니다.

## 첫 범위에서 명시적으로 제외할 내용

- 살아 있는 유명인
- 사용자가 직접 입력하는 유명인 이름
- 네 번째 독자 또는 모델 호출 예산 증가
- 모델이 선택하는 게스트
- 런타임 인물 자료 검색
- 목소리 흉내 또는 생성된 역사적 인용문
- 첫 전체 mock 평가 전에 두 명 이상의 게스트 구현
