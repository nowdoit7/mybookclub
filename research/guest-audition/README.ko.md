# 역사적 게스트 오디션 랩

이 연구용 랩은 역사적·문학적 상상 속 게스트를 실제 제품에 추가하기 전에 독서
대화를 더 좋게 만드는지 검증합니다. 제품 API 키를 사용하지 않으며 세션 진행
엔진도 변경하지 않습니다.

- [1차 오디션 결과](./ROUND_01_RESULTS.ko.md)
- [장르군별 후보 대기열](./CANDIDATE_QUEUE.ko.md)

## 방법

각 사례는 도서 근거, 토론 주제, 사용자 주장, 발언 순서와 횟수를 고정합니다.
`sample-a`와 `sample-b`는 동일한 카테고리 독자를 교체합니다. 하나는 현재 일반
독자 기준선이고 다른 하나는 상상 속 게스트지만, 블라인드 평가가 끝날 때까지
어느 쪽인지 평가자에게 알리지 않습니다.

사례 스키마는 두 샘플 사이에서 후보 이외의 발언, 후보의 위치, 발언 순서가 달라지면
거부합니다. 평가 근거 ID도 해당 샘플에 실제로 존재하는 발언만 허용하므로 비교 과정을
다시 감사할 수 있습니다.

서로 독립적인 평가자 세 명이 두 샘플에 1~5점을 주고 근거 발언 ID를 기록합니다.
블라인드 선호를 확정한 다음에만 게스트를 공개하고, 가짜 인용, 날조된 전기,
현대 책을 실제로 읽었다는 주장, 권위 효과, 다른 책의 정보 유출, 진행 규칙 위반을
레드팀 방식으로 검사합니다.

집계 명령은 다음과 같습니다.

```bash
npm run evaluate:guest-audition
```

## 확인할 수 있는 것

- 해당 책에 맞는 새로운 해석을 추가하는가
- 강의하지 않고 대화에 참여하는가
- 유명한 이름을 가려도 자연스럽고 흥미로운가
- 한 작품에 오버핏되거나 다른 책의 내용이 유출되지 않는가
- 실제 구현과 Live 테스트를 진행할 가치가 있는가

역사적 정확성, 법적 안전성, 실제 사용자의 권위 편향, 제품 GPT-5.6 호출 지연은
이 랩만으로 인증할 수 없습니다. 후보가 통과한 뒤 검토된 출처, 사람의 검토,
마지막 Live 스모크 테스트가 필요합니다.

## 통과 기준

- 즉시 실패 0건
- 게스트 평균 4.0/5 이상
- 책 구체성, 새로운 해석, 대화 참여도 각각 4.0 이상
- 자연스러움과 권위 균형 각각 3.5 이상
- 블라인드 게스트 선호율 65% 이상
- 기준선보다 새로운 해석 점수 +0.5 이상
- 기준선보다 자연스러움 하락 폭 -0.25 이내
- 모든 후보 발언 2~4문장

생성된 정확한 문장 자체를 테스트 정답으로 사용하지 않습니다. JSON 결과에는
근거, 점수, 구조적 속성만 보존합니다.
